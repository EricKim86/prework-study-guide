HTML

  • HTML: Hyper Markup Language
  • Element: Type of content on a webpage. Common examples include text, image, header, and list elements.
  • HEAD element contains information, or metadata, that helps the browser to render the page correctly.
  • BODY element contains the content that is shown to the user.
  • The "id" attribute is a way to identify elements and select them.
  • The "ul" tag indicates an unordered list.

CSS

  • CSS: Cascading Style Sheets
  • CSS can be added to HTML in three ways: Inline, Internal, and External.
  • An inline CSS is used to apply a unique style to a single HTML element.
  • An internal CSS is used to define a style for a single HTML page.
  • An external CSS is used to define the style for many HTML pages.
  • Some common CSS properities include "color", "font-family", and "font-size".
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git clone [url]: retreive an entire repository from a hosted location via URL.
  • git branch: list your branches
  • git branch [branch-name]: create a new branch at the current commit.
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot.
  • git merge [branch]: merge the specified branch's history into the current one.
  • git revert: recert some existing commits.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is a single variable that is used to hold a group of Data.
  • The "console.log" method can be used to output a message to the web console by adding an argument.
  • A function does not automatically execute when the JavaScript file is run. You must call a function by name.
